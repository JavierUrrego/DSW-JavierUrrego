Contenido

GIT: Simplifica el proceso de trabajo con otras personas y facilita la colaboración en proyectos, a su vez es un sistema de control de versiones, nos ayuda a gestionar nuestros archivos de proyecto los principales beneficios que tiene GIT es que nos permite tener un registro de toda la historia de los proyectos en los cuales estamos trabajando siendo parte fundamental de los desarrolladores porque cuando se está trabajando en un proyecto primero se construye una versión básica del mismo y luego se busca mejorarlo añadiendo nuevas características o simplemente experimentando con el fin de mejorar el producto, el control de versiones realiza un seguimiento automático de cada cambio y permite volver a una versión anterior sin importar cuantas veces allá cambiado sus archivos, otro de los grandes beneficios que tenemos con GIT es que permite a las personas trabajar juntos en el mismo proyecto al mismo tiempo sin perturbar los archivos de los demás logrando que los miembros del equipo trabajen en diferentes características fusionando fácilmente los cambios. GITHUB: Es un servicio basado en la web para el control de versiones usando GIT. Básicamente, es un sitio de redes sociales para desarrolladores logrando mirar el código de otras personas, esta plataforma nos permite publicar nuestros proyectos para presentar a todo aquel que esté interesado en nuestro proyecto bien sea un potencial empleador, o la comunidad desarrolladora siendo una plataforma web para el desarrollo de versiones.

Diferencias

  • GIT: herramienta de control de versiones distribuida que puede gestionar el código fuente de un proyecto de desarrollo, ademas es una herramienta de código abierto que los desarrolladores instalan localmente para gestionar el código fuente.
  • GITHUB: plataforma basada en la nube costruida alrrededor de la herramienta GIT que presta un servicio en línea al que los desarrolladores que utilizan GIT pueden conectarse y cargar o descargar recursos.
